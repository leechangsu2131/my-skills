## 개요

나이스(NEIS)는 신청 화면을 팝업창으로 열고 보안 프로그램이 강력하게 동작하므로, 아래와 같이 **사용자가 직접 로그인 → 셀레니움이 이후 작업 인계** 방식을 사용합니다.

---

## 준비 사항

``` 
파일 위치: skills/neis-leave-bot/
  ├── launch_chrome.bat   # 크롬 실행기 (원격 디버깅 포트 9222)
  ├── neis_test.py        # 구조 파악 + 팝업 탐색 테스트 스크립트
  └── neis_apply.py       # (향후) 실제 연가/출장 자동 신청 스크립트
```

### 라이브러리 설치 (최초 1회)
```
pip install selenium webdriver-manager
```

---

## 사용 방법

### 1단계: 특수 크롬 실행
`launch_chrome.bat` 을 더블클릭하여 실행합니다. 일반 크롬과 별도로 나이스 창이 열립니다.

> ⚠️ 반드시 이 방법으로 실행한 크롬을 사용해야 셀레니움이 붙을 수 있습니다.  
> 평소에 쓰던 크롬 창과는 **별개의 창**입니다.

### 2단계: 직접 로그인
열린 크롬에서 나이스에 **수동으로 로그인**합니다.  
(공동인증서, 보안 프로그램 등 평소대로 진행)

### 3단계: 테스트 스크립트 실행
로그인 완료 후 터미널에서 실행합니다.

```bash
cd skills/neis-leave-bot
python neis_test.py
```

**스크립트 동작 순서:**
1. 열려있는 크롬에 셀레니움 연결 (attach)
2. 창 목록(탭/팝업) 출력
3. 현재 창의 폼 요소(입력칸) 분석 출력
4. **[중요]** "지금 메뉴를 클릭하여 팝업창을 열어보세요" 출력 후 20초 대기
   - 이때 나이스에서 **연가 신청** 또는 **출장 신청** 메뉴를 직접 클릭
5. 팝업창을 셀레니움이 자동 감지하여 전환
6. 팝업창 내 폼 요소 + 버튼 목록 출력

### 4단계: 결과 분석 및 자동 신청 코드 제작
`neis_test.py` 실행 결과(터미널 출력)를 챗봇에 붙여넣으면, 결과에서 파악한 요소들(id, name 등)을 이용하여 `neis_apply.py` 자동 신청 스크립트를 즉시 제작해 드립니다.

## 자주 발생하는 문제

| 증상 | 해결책 |
|------|--------|
| `크롬 연결 실패` 오류 | `launch_chrome.bat` 으로 먼저 크롬을 실행했는지 확인 |
| 크롬이 이미 실행중이라 포트 충돌 | 기존 크롬 창을 모두 닫고 `launch_chrome.bat` 재실행 |
| 팝업창 감지 실패 | 팝업 차단 설정 해제 후 재시도, 또는 직접 팝업 클릭 후 5초 내로 스크립트 재실행 |
| iframe 접근 실패 (보안) | 같은 도메인 내의 iframe만 접근 가능, 나이스 보안정책에 따라 일부 제한될 수 있음 |
