# 매매일지 자동화 워크플로우

1. 사용자가 "/record-trade [자연어 매매 기록]" 형식으로 명령어를 입력합니다. (예: "/record-trade 오늘 SCHD 10주를 41,000원에 매수함. 배당락일 전이라 포모 와서 샀음. 만족도는 6점")
2. AI는 입력된 자연어를 분석하여 다음 14개의 열(Column) 데이터 배열로 완벽하게 매핑합니다. 빈 값이 있다면 빈 문자열("")로 처리합니다:
   - A(매매일): YYYY. MM. DD 형식 (제공되지 않으면 오늘 날짜 사용)
   - B(종목티커): (예: SCHD)
   - C(종목명): (예: Schwab US Dividend Equity ETF)
   - D(매매 구분): "매수" 또는 "매도"
   - E(수량): 숫자 (예: "10")
   - F(단가): 숫자 (콤마 포함, 예: "41,000")
   - G(금액): 수량 * 단가 (콤마 포함, 자동 계산, 예: "410,000")
   - H(매매 이유): 왜 이 종목을 샀는지/팔았는지
   - I(타이밍 이유): 왜 하필 지금 샀는지/팔았는지
   - J(만족도): 1~10점 사이 점수
   - K(사후분석): 공란
   - L(컨디션): 당시 심리/건강 상태
   - M(인지오류): 인지적 편향 (예: 앵커링효과, 이익확정, 손실회피, 추세추종, 포모, 군중심리 등 사용자 시트에 있는 항목 중 맥락에 맞는 것 선택)
   - N(해결전략): 인지오류에 대한 피드백이나 다짐
3. 위에서 추출된 값을 JSON 형태의 `--json` 파라미터로 조합합니다. 쌍따옴표 등 이스케이핑에 주의하여 올바른 JSON 리터럴 문자열로 생성해야 합니다.
4. 아래 명령어를 통해 구글 시트 "매매일지" 탭에 데이터를 추가합니다.
// turbo
5. 명령 실행:
```bash
gws sheets spreadsheets values append \
  --params '{"spreadsheetId": "1pgnG2bepfpwA0PY43LXaU1FSSF8U7qe8Cl_r2QvJgdU", "range": "매매일지!A:N", "valueInputOption": "USER_ENTERED"}' \
  --json '{"values": [["<A>", "<B>", "<C>", "<D>", "<E>", "<F>", "<G>", "<H>", "<I>", "<J>", "<K>", "<L>", "<M>", "<N>"]]}'
```
6. 터미널 명령이 완료되면, 사용자에게 시트에 기록된 내용을 예쁜 마크다운 표 형식으로 정리해서 보고합니다.
