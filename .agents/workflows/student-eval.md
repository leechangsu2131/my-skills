# 학기말 평가 문장 생성 워크플로우

노션에 축적된 학생별 기록을 기반으로 교과 평가 또는 종합발달상황 문장을 자동 생성합니다.

## 사전 조건

- 노션 MCP 연동이 되어 있어야 합니다
- 노션에 학생 기록 DB가 있고, 기록이 축적되어 있어야 합니다
- 평가 유형(교과평가/종합발달상황)을 선택할 수 있어야 합니다

---

## Step 1: 대상 학생 및 평가 유형 선택

사용자에게 다음을 확인합니다:

```
어떤 학생의 평가를 작성할까요?
- 특정 학생 (예: "김민준")
- 전체 학생 일괄 생성

평가 유형은요?
- 교과 평가 (과목 선택)
- 종합발달상황
- 전체 (교과 + 종합)
```

## Step 2: 노션에서 학생 기록 조회

노션 학생 기록 DB에서 해당 학생의 전체 기록을 조회합니다:

1. 학생 기록 DB에서 해당 학생 이름으로 필터링
2. 날짜순 정렬
3. 전체 기록 불러오기
4. 분야별, 과목별로 정리

---

## Step 3: 기록 분석 요약 보기

기록 분석 요약을 사용자에게 보여줍니다:

```markdown
## 김민준 학생 기록 분석

📊 총 23건 (2026-03-02 ~ 2026-07-15)
😊 긍정 18건 | 😐 중립 3건 | 😟 부정 2건

과목별: 수학 8건, 국어 5건, 과학 4건, 기타 6건
```

사용자가 추가 정보를 입력할 수 있습니다:
- "체육 기록도 이거 추가해줘: ..."
- "리더십을 강조해줘"

---

## Step 4: 평가 문장 생성

`@student-eval-generator` 스킬의 가이드라인에 따라 문장을 생성합니다:

1. 3인칭 관찰자 시점
2. 긍정 우선, 성장 변화 강조
3. 구체적 근거(실제 기록) 기반
4. 적절한 길이 (교과별 2~4문장, 종합 5~8문장)

생성된 문장을 사용자에게 보여줍니다.

---

## Step 5: 수정 및 확정

사용자의 수정 요청을 반영합니다:

```
수정할 부분이 있나요?
- "더 짧게"
- "리더십 부분 강조"
- "부정 표현 빼줘"
- "성취기준 ○○에 맞춰줘"
- "괜찮아, 저장해줘" → 노션에 저장
```

## Step 6: 노션에 저장 (선택)

확정된 문장을 노션 평가 초안 DB에 저장합니다:

1. 평가 초안 DB에 학생명 + 평가유형 + 과목 + 문장 저장
2. 이미 같은 학생의 초안이 있으면 업데이트 여부 확인

```
✅ 평가 초안 저장 완료!
- 김민준: 수학 교과평가 + 종합발달상황
- [평가 초안 DB 바로가기]
```

---

## 전체 학생 일괄 처리 시

전체 학생을 선택한 경우:

1. 학생 번호순으로 한 명씩 생성
2. 각 학생마다 간단한 미리보기 출력
3. 모두 완료 후 일괄 노션 저장
4. 기록이 부족한 학생은 표시 (⚠️ "기록 3건 미만")

---

## 주의사항

- AI가 생성한 문장은 **반드시 선생님이 검토** 후 사용하세요
- 기록이 적은 학생은 문장이 빈약할 수 있습니다 — 추가 관찰 내용을 입력해 주세요
- 성취기준 PDF를 프로젝트에 업로드하면 더 정확한 문장 생성이 가능합니다
