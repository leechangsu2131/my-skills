# Discord 메모/공지 요약 에이전트 스킬

디스코드 채널을 개인 메모장 겸 정보 저장소로 사용하고, 안티그래비티가 이를 정기적으로 검토하여 체계적인 업무 공지/요약 리포트를 작성해 다른 채널로 보내주는 워크플로우입니다.

## 개요 (수동/대화형 방식)

파이썬 기반 자동 봇 대신, **사용자가 원할 때 언제든 안티그래비티에게 요약을 지시**하여 고품질의 문맥(Context) 파악 요약을 받아보는 방식입니다.

```
[사용자] 입력 채널에 잡다한 업무 메모, 링크, 할일 입력
    ↓
[사용자] "디스코드 요약해서 공지 올려줘" 라고 채팅 입력
    ↓ (API 혹은 MCP로 내용 읽기)
[안티그래비티] 메시지 내용 정독 → 업무 맥락에 맞춘 구조화된 요약문 작성
    ↓ (Webhook 전송)
[출력 채널] 깔끔하게 정리된 업무 공지/요약 리포트 전송
```

## 확인된 환경 정보 (.env 및 시스템 설정)

| 항목 | 값 | 역할 |
|------|----|------|
| **입력 (메모 채널)** | `1477078755951378432` | 사용자가 평소 메모를 남기는 곳 (읽기 대상) |
| **출력 (웹훅 URL)** | `https://discordapp.com/api/webhooks/1477089851928084664/C7DYM4WEuNueIj5uDihcQt6Js2rm9FtiDE1G3t7YxBuxFJolVlZP0epsbUEPS58Hk_Bq` | 안티그래비티가 요약본을 쏴주는 공지 채널 |
| **봇 접근 (Token)** | `DISCORD_BOT_TOKEN` | (환경변수/Python 스크립트 기반 조회 시 사용) |

> ⚠️ 봇 권한 중 **Message Content Intent** 설정이 켜져 있어야 메시지 원문을 정상적으로 읽을 수 있습니다. (현재 활성화 완료 ✅)

## 🚀 안티그래비티에게 지시하는 방법 (주요 프롬프트)

사용자는 생각날 때마다 아래 명령을 활용하여 안티그래비티에게 작업을 시킵니다.

### 1단계: 업무 요약 리포트 작성 후 발송
```text
"디스코드 입력 채널(1477078~...) 메시지를 읽어봐.
내용을 싹 분석해서 학교 업무용 공지 형태로 보기 좋게 구조화한 뒤,
출력 웹훅을 통해 요약 리포트로 전송해줘."
```
👉 안티그래비티 행동 지침:
1. `requests`나 MCP 도구를 이용해 지정된 채널의 최근 메시지를 읽어옵니다. (단순 개수뿐 아니라 실제 텍스트 원문 취합)
2. 단순한 나열이 아닌, "학교 업무" 등의 맥락을 파악하여 **공유 됨/건의 됨/참고 할 것** 등의 실무형 리포트로 정리합니다.
3. 대상 Webhook URL을 향해 `{"content": "..."}` JSON 페이로드를 POST 요청으로 전송합니다.

### 2단계: 할일/일정 감지 및 구글 캘린더 등록
```text
"방금 읽은 디스코드 메모 중에 일정이나 마감 기한이 있는 할 일이 있으면 찾아서 구글 캘린더에 바로 등록해줘."
```
👉 안티그래비티 행동 지침:
1. 최근 읽어온 디스코드 메시지 텍스트를 분석해 날짜, 시간, 행위를 도출합니다.
2. 구글 캘린더 MCP 또는 API를 사용하여 해당 일정을 사용자의 캘린더에 바로 꽂아 넣습니다.
3. 등록 완료 결과를 알려줍니다.

## 🛠️ (참고) Python 스크립트 수동 검사/전송

요청에 따라 안티그래비티가 환경에 구축해둔 파이썬 스크립트 라인업입니다.
위의 대화형 지시 대신 직접 콘솔에서 스크립트를 변경해가며 테스트할 때 활용합니다. (위치: `skills/discord-memo-agent/scripts/`)

- `test_content.py`: 현재 입력 채널의 메시지 원문(내용)이 제대로 봇 권한으로 읽히는지 확인하고 `.json`으로 덤프하는 스크립트.
- `test_school_summary.py`: 현재 `messages.json`에 저장된 원문을 바탕으로 안티그래비티가 작성한 '학교 업무용 구조화 공지' 텍스트를 웹훅으로 전송하는 스크립트.
- `agent.py`: 백그라운드에서 일정 주기별로 작동하게 만들 수 있는 자동화 에이전트. (현재 시스템 상 요약 품질 문제로 수동 방식을 추천)

## 작업 시 주의사항 (안티그래비티 지침)

1. 사용자가 요약을 요청하면, 절대 수량("메시지 3건", "링크 1개")만 나열하지 마세요. 
2. 무조건 모든 원본 메시지의 텍스트를 읽고 문맥(예: 학교 정보, 업무, 영수증, 구매 등)을 통째로 이해하여 **카테고리화, 구조화된 리포트**를 생성해야 합니다.
3. 디스코드에서 불러온 빈 텍스트(" ")나 시스템 메시지는 걸러내고 유효한 텍스트만 집계하세요.
