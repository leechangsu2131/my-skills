# 경주시립도서관 재고 검색 및 희망도서 자동 신청 봇 📚🤖

이 프로젝트는 **Notion 도서 데이터베이스**와 연동하여 사고 싶은 책(또는 읽고 싶은 책)의 **도서관 소장 여부를 자동으로 파악**하고, 도서관에 없는 책은 **희망도서로 자동 신청**해주는 RPA(Robotic Process Automation) 스크립트 모음입니다.

## ✨ 핵심 기능

1. **교보문고 데이터 파싱 & ISBN 추출 (`add_and_sync_books.py`)**
   - 노션에 입력된 '책 제목(및 저자)'을 바탕으로 교보문고를 정확도 높게 검색
   - 올바른 **ISBN-13**과 **서점 상품 페이지 URL**을 추출하여 노션 DB에 자동 기입
2. **도서관 소장 여부 확인 및 지점 태그 자동화 (`add_and_sync_books.py`)**
   - 추출한 ISBN을 통해 경주시립도서관 서버에서 소장 여부 검사
   - 소장 중이라면 해당 지점(`시립`, `송화`, `단석`, `칠평`, `전자`)을 노션 태그로 반영
   - 미소장 시 `미소장` 태그 적용
3. **희망도서 100% 무인 자동 신청 (`request_books.py`)**
   - 노션에서 `미소장` 태그가 달린 책들을 찾아 경주시립도서관 웹사이트에 Playwright를 이용해 자동 신청 (우선 지점: **송화도서관**)
   - **강력한 자동화 로직**: 
     - "개인정보 수집 및 이용 동의" 체크박스 자동 처리 (별도 수동 조치 불필요)
     - 1인 월 2권 한도 도달 시 `.env`에 등록된 **다음 계정으로 실시간 자동 전환(Rotation)**
     - "이미 도서관에 소장 중인 자료", "이미 누군가 신청 중인 자료" 등 도서관 서버의 다양한 팝업 반려 사유를 완벽히 감지하여 무시하고 다음 책으로 넘어감
   - 성공 시 노션 DB의 `신청현황` 컬럼을 `송화도서관 신청`으로 업데이트

## ⚙️ 필요 조건 및 설치

- Python 3.8 이상 필요
- `requests`, `beautifulsoup4`, `playwright` 라이브러리 필요
- 노션 API 통합(Integration) 토큰 및 연결

```bash
# 필수 라이브러리 설치
pip install -r requirements.txt

# 희망도서 신청용 Playwright 브라우저 필수 설치
playwright install chromium
```

---

## 🛠️ 설정 방법 (환경 변수)

1. 폴더 내의 `.env.example` 파일을 복사하여 `.env` 파일을 생성합니다.
```bash
cp .env.example .env
```
2. 생성된 `.env` 파일에 발급받은 노션 데이터 및 경주시립도서관 계정 정보를 입력합니다.
   - 가족/지인 계정 등 **도서관 계정을 여러 개 등록하면(예: `LIBRARY_ACCOUNT_1_ID`, `LIBRARY_ACCOUNT_2_ID`...) 계정당 월 2권 제한을 뚫고 끝없이 신청할 수 있습니다.**

## 🚀 사용 가이드

### 1단계: 노션 DB 업데이트 (검색 및 소장 확인)
노션에 오늘 추가한 책들의 소장 정보와 ISBN을 일괄 채우고 싶을 때 실행합니다.
```bash
python add_and_sync_books.py
```

### 2단계: 희망도서 자동 신청 (매크로 구동)
`미소장` 도서들에 대해 일괄적으로 도서관에 희망도서 신청을 넣습니다.
```bash
# [추천] 먼저 실제 신청을 누르지 않고 폼 작성까지만 시뮬레이션 해보기
python request_books.py --dry-run

# 실제 신청 진행! 한 달에 한 번씩 돌려주면 됩니다.
python request_books.py
```

### 🎯 팁: 특정 책만 골라서 신청하고 싶을 때
'미소장' 태그 여부에 상관없이, 지금 당장 특정 책 1권만 바로 신청해보려면 `--title` 옵션을 사용하세요.
```bash
python request_books.py --title "세컨드 브레인"
```

## 🚨 유의 사항
- 기본적으로 신청 지점은 `송화도서관`을 최우선으로 선택하도록 하드코딩 되어 있습니다.
- 교보문고 검색 시 동명이도서(제목은 같으나 다른 책)가 잡히는 것을 최소화하기 위해 **노션에 '저자' 컬럼도 꼼꼼히 적어두시면 파싱 정확도가 훨씬 올라갑니다.**
- `library_notion_sync.py` 및 `resync_14books.py`는 과거 구조 개편 시 사용된 특수목적 스크립트이므로 무시하셔도 무방합니다.
