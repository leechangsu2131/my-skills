**1장. 설계 원칙**

**1.1 이 시스템이 해결하는 문제**

\"저 물건 어디 있지?\"라는 질문에 5초 안에 답하는 것. 그 이상도 이하도
아닙니다.

물건을 찾는 데 시간이 걸리는 이유는 두 가지입니다. 첫째, 어디 뒀는지
기억이 나지 않는다. 둘째, 기억은 나는데 실제로 거기 없다(누군가 옮겼거나
내가 잘못 기억하고 있다). 이 시스템은 두 가지 문제를 모두 해결합니다.

**1.2 우선순위**

  ---------- ------------------ ----------------------------------------------
  **순위**   **기능**           **설명**

  1          물건 찾기 (검색)   이름으로 검색 → 정확한 위치 경로 반환.
                                시스템의 존재 이유.

  2          위치 등록·수정     물건을 어디에 뒀는지 빠르게 등록. 30초 이내.

  3          공간 구조 관리     방·가구를 계층으로 정의. 처음 한 번 세팅하면
                                이후 거의 변경 없음.

  4          2D 평면도 시각화   공간을 2D 평면도로 보며 위치 파악. Phase 1부터
                                바로 구현. 3D는 필요 없음.

  5          대여·인수인계 추적 교실 등 공용 공간용 확장 기능. 현재 범위 외,
                                추후 플러그인 방식으로 추가.
  ---------- ------------------ ----------------------------------------------

**1.3 범용 설계 원칙 (5가지)**

- 단일 진실 공급원: 물건 위치는 오직 이 시스템에만 존재한다.
  포스트잇·수첩 병행 금지.

- 최소 입력: 등록 시 필수 입력은 이름·위치 2가지뿐이다. 나머지는 선택.

- 공간 계층 존중: 공간 → 가구 → 구획의 3단계를 항상 유지한다. (2D
  기준으로 L2 구역 생략 가능)

- 변경 이력 보존: 물건의 위치가 바뀔 때 이전 위치를 지우지 않고 기록으로
  남긴다.

- 컨텍스트 독립: 집에서도, 교실에서도, 사무실에서도 같은 구조로 쓸 수
  있어야 한다.

**1.4 사용 컨텍스트별 적용 방식**

  --------------- ------------------------- ---------------------------------
  **컨텍스트**    **공간 예시**             **특이사항**

                                            메모 필드 활용.

  교실            교탁·교구장·사물함·창가   교사 단독 관리. 대여·인수인계
                  선반                      기능은 추후 확장.

  사무실 (확장)   책상·공용 캐비닛·창고     동일 구조 적용 가능. 별도
                                            인스턴스 생성 권장.

**2장. 공간 계층 구조**

**2.1 3단계 계층 (2D 기준)**

2D 평면도 시스템에서는 3단계 계층을 기본으로 사용한다. L2 구역은 선택
사항이며, 공간이 단순한 경우 공간 → 가구 → 구획의 3단계로 충분하다.

  ---------- --------------- ---------------------- -----------------------
  **레벨**   **명칭**        **집 예시**            **교실 예시**

  L1         공간 (Space)    우리 집                3학년 2반

  L2 (선택)  구역 (Area)     거실, 침실, 주방       앞쪽, 뒤쪽, 창가

  L3         가구            TV 장식장, 옷장, 책장  앞 교구장, 사물함, 교탁
             (Furniture)                            

  L4 (선택)  구획 (Zone)     1번 서랍, 상단 칸      1단 서랍, 상단 오른쪽
  ---------- --------------- ---------------------- -----------------------

> *💡 2D 평면도에서는 가구를 직사각형 마커로 표시하고, 클릭 시 보관 물건
> 목록을 확인한다. L2 구역은 방이 여러 개이거나 공간이 복잡할 때만
> 추가한다.*

**2.2 공간(L1) 설정 기준**

- 물리적으로 분리된 장소를 각각 하나의 공간으로 등록한다.

- 예: 집과 교실을 동시에 관리한다면 두 개의 공간으로 분리한다.

- 공간 이름은 고유해야 한다. \"우리 집\", \"3학년 2반\"처럼 특정할 수
  있는 이름 사용.

**2.3 가구(L3) 배치 가이드**

2D 평면도에서 가구는 실제 배치를 반영하여 드래그로 위치를 조정한다.

  ------------------ ------------------------ ----------------------------
  **구분 기준**      **집 적용 예**           **교실 적용 예**

  기능               수납·취침·요리·공부 공간 수업준비·보관·청소 구역

  물리 위치          창가 쪽, 입구 쪽         앞쪽, 뒤쪽, 복도 쪽, 창가 쪽

  사용 빈도          자주 쓰는 물건 / 계절    매일 쓰는 것 / 가끔 쓰는 것
                     용품                     
  ------------------ ------------------------ ----------------------------

> *💡 가구 개수: 집은 4\~8개, 교실은 4\~7개가 적당하다. 2D 평면도가
> 복잡해지면 오히려 불편하다.*

**3.1 물건(Item) --- 핵심 스키마**

★ 표시는 필수. 나머지는 있으면 좋은 선택 사항. 처음에는 필수만으로

  --------------- ---------- ---------- ------------------------------------------
  **필드명**      **타입**   **필수**   **설명**

  name            String     ★          물건 이름. 최대 60자. 나중에 검색할 때 쓸
                                        이름으로 직관적으로 짓는다.

  furniture_id    FK         ★          어느 가구인지 (L3). 가구 목록에서 선택.

  category        Enum       ---        분류. 교실: 교구·문구·전자기기·서류·기타 /
                                        집: 전자기기·의류·생활용품·서류·공구·기타

  quantity        Int        ---        수량. 기본값 1.

  context         Enum       ---        사용 컨텍스트. 집 / 교실 / 공통.

  created_at      Date       ★          최초 등록 일시. 자동 입력.

  updated_at      Date       ★          마지막 수정 일시. 자동 갱신.

**4장. 시스템 규칙**

**4.1 명명 규칙**

**R-N-01 \| 가구 이름**

- 위치 + 종류 형식 권장. 예: \"TV 장식장\", \"침실 옷장\", \"앞 교구장\"

- 같은 공간에 동일 종류가 여럿이면 번호 추가. 예: \"교구장 1\", \"교구장
  2\"

- 최대 30자

**R-N-02 \| 구획 이름**

- 방향·순서 기준: 위→아래, 왼→오른 순서로 번호 부여

- 예: \"1번 서랍\", \"상단 칸\", \"오른쪽 2단\", \"도어 포켓 위\"

- 최대 20자

**R-N-03 \| 물건 이름**

- 일반 명사 우선. 브랜드명은 구분에 필요할 때만 포함.

- 동일 이름 물건이 여럿이면 구분자 추가: \"가위 (빨강)\", \"가위
  (파랑)\"

- 나중에 내가 뭐라고 검색할지를 기준으로 짓는다.

> *⚠️ 물건 이름에 위치 정보를 넣지 않는다. \"서랍 속 가위\" ❌ → 위치는
> 위치 필드에, 이름은 이름답게.*

**4.2 등록 규칙**

**R-D-01 \| 위치 단일성**

물건은 항상 정확히 하나의 위치를 가진다. \"어딘가에 있음\"은 허용되지
않는다. 위치가 불확실하면 memo 필드에 \"위치 확인 필요\"를 기재하고
마지막으로 본 가구에 등록해 둔다.

**R-D-02 \| 물건 이동 시**

1.  위치가 바뀌었을 때: 2D 평면도에서 새 가구를 선택하거나 물건 편집으로
    furniture_id를 변경

2.  History 시트에 이전 위치 자동 기록 (언제 어디서 어디로 이동했는지)

3.  이전 레코드는 삭제하지 않는다.

> *💡 2D 평면도에서 물건을 다른 가구로 이동시킬 때도 동일 규칙 적용.*

**R-D-03 \| 삭제·분실 처리**

- 물건을 버리거나 잃어버린 경우: 레코드를 삭제하지 않고 memo에 \"폐기
  2026-03\" 또는 \"분실 추정\" 기재

- 나중에 같은 물건을 다시 살 때 이전 기록을 참고할 수 있다.

**4.3 사진 등록 기준**

- 가구 사진: 전면 또는 내부 구획이 잘 보이는 사진 1장 권장. 2D
  평면도에서 클릭 시 참고용으로 표시된다.

- 물건 사진: 동일 이름 물건이 2개 이상일 때 구분용으로 권장.

- 사진 없어도 시스템은 완전히 동작한다. 사진은 시각적 확인 수단일
  뿐이다.

**5장. 검색 규칙**

**5.1 검색 우선순위**

검색어 입력 시 아래 순서로 일치 여부를 판단하고, 상위 순위 결과를 먼저
보여준다.

  ---------- ------------------ ----------------------------------------------
  **순위**   **대상**           **설명**

  1          이름 완전 일치     검색어 = 물건 이름 전체. 최우선 표시.

  2          이름 부분 일치     검색어가 이름에 포함됨.

  3          태그 일치          검색어가 tags 배열에 포함됨.

  4          메모 포함          검색어가 memo 필드에 포함됨. 가장 낮은
                                우선순위.
  ---------- ------------------ ----------------------------------------------

**5.2 검색 결과 표시**

- 결과는 \"공간 \> 가구 \> 구획\" 전체 경로로 표시한다.

- 2D 평면도에서 해당 가구를 하이라이트(테두리 강조)하여 시각적으로
  안내한다.

- 검색 결과 클릭 시 해당 공간으로 자동 이동하고 가구가 선택된다.

**7장. 2D 평면도 운용 가이드**

**7.1 2D 평면도 설계 원칙**

2D 평면도는 실제 공간의 구조를 직관적으로 반영해야 한다. 복잡한 3D 모델
없이도 \"저 물건 어디 있지?\"를 5초 안에 해결하는 것이 목표다.

- 가구는 실제 배치를 반영하여 평면도 위에 배치한다. 드래그로 자유롭게
  조정 가능.

- 가구 크기(가로·세로)는 실제 비례를 대략 맞추면 직관성이 높아진다.

- 가구 이름은 평면도 마커 안에 표시되므로 짧고 명확하게 짓는다.

**7.2 단계별 세팅 가이드**

  ---------- ------------------ -------------------------------------------
  **단계**   **목표**           **작업**

  Step 1     공간 등록          관리할 공간 이름 등록 (예: 3학년 2반, 우리
                                집)

  Step 2     가구 배치          실제 배치를 참고해 가구 마커를 평면도에
                                추가하고 드래그로 위치 조정

  Step 3     물건 등록          가구를 클릭 → 물건 추가. 이름·분류·수량
                                입력. 30초 이내 목표.

  Step 4     검색 테스트        \"저 물건 어디 있지?\" 검색으로 5초 안에
                                찾을 수 있는지 확인

  Step 5     구획 추가          서랍·칸이 여러 개인 가구는 구획(Zone)을
  (선택)                        추가해 정확도 향상
  ---------- ------------------ -------------------------------------------

**7.3 가구 마커 색상 가이드**

마커 색상은 가구 유형이나 구역을 구분하는 데 활용한다. 색상에 규칙을
붙이면 평면도를 훑는 속도가 빨라진다.

  ------------ ---------------------- ------------------------------------
  **색상**     **권장 용도**          **예시**

  보라 계열    교육·교구 관련 가구    앞 교구장, 뒤 교구장

  초록 계열    전자기기 관련 가구     컴퓨터 책상, 충전 스테이션

  주황 계열    수납·생활용품 가구     사물함, 수납 박스

  파랑 계열    창가·개방형 선반       창가 선반, 오픈 책장

  노랑 계열    개인 물품 보관 가구    개인 사물함, 서랍장
  ------------ ---------------------- ------------------------------------

> *💡 색상은 강제 규칙이 아니다. 팀(교실)에서 함께 정하면 공유 공간
> 관리가 더 쉬워진다.*

**9장. 용어집**

  ---------------------- ------------------------------------------------
  **용어**               **정의**

  공간 (Space, L1)       관리 대상의 최상위 단위. 물리적으로 분리된 장소.
                         예: 우리 집, 3학년 2반.

  구역 (Area, L2)        공간 내 기능·위치로 구분되는 구역. 2D 기준에서는
                         선택 사항. 예: 거실, 교탁 구역.

  가구 (Furniture, L3)   수납 기능이 있는 물리적 가구나 설비. 2D
                         평면도에서 마커로 표시. 예: TV 장식장, 앞
                         교구장.

  구획 (Zone, L4)        가구 내 물리적으로 구분되는 최소 단위. 서랍 1개,
                         선반 1단. 이 단위까지 지정해야 \"바로 찾기\"가
                         실현된다.

  2D 평면도              공간을 위에서 내려다본 평면 지도. 가구를
                         직사각형 마커로 표시하고, 클릭 시 물건 목록을
                         확인한다.

  위치 경로 (Path)       \"공간 \> 가구 \> 구획\" 형식의 전체 위치
                         문자열. 검색 결과에 항상 표시.

  하네스 엔지니어링      구현 전에 규칙·데이터 모델·평가기준을 문서화하여
                         개발 방향을 정렬하는 설계 방법론. 이 문서가 그
                         산출물.

  단일 진실 공급원       동일한 정보가 여러 곳에 분산되지 않도록 하는
                         원칙. 물건 위치는 이 시스템 하나에만 존재한다.
