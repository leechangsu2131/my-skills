# 브릿지 시트 설계안

## 목적

현재 구조에서는 진도표 시트의 `계획일`이 다음 역할을 동시에 맡고 있다.

- 수업 순서 결과 저장
- 날짜별 실제 수업 배치 저장
- 오늘/내일/다음 주 조회 기준
- 완료 처리 대상 찾기 기준

이 구조는 단순하지만, 진도표의 수업 내용을 수정하거나 차시 순서를 바꾸면 날짜 매핑을 다시 수행해야 하는 결합도가 크다.

이를 줄이기 위해 데이터를 아래처럼 분리한다.

- `진도표`: 무엇을 가르칠지 관리
- `브릿지 시트`: 언제 가르칠지 관리
- `기초시간표/휴업일/수업시작일`: 자동 배치 규칙 관리

## 목표 구조

### 1. 진도표 시트

직접 수정하는 본문 데이터다.

권장 컬럼:

- `lesson_id`
- `수업내용`
- `과목`
- `대단원`
- `차시`
- `실행여부`
- `계획일`
- `계획교시`
- `pdf파일`
- `시작페이지`
- `끝페이지`
- `비고`
- `연기횟수`
- `소단원`

설명:

- `lesson_id`는 브릿지 시트와 연결하기 위한 고정 식별자다.
- `계획일`, `계획교시`는 당분간 호환용 표시 컬럼으로 유지한다.
- 사람이 직접 수정하는 기준은 `수업내용`, `과목`, `대단원`, `차시`, `실행여부` 쪽이다.

### 2. 브릿지 시트

날짜별 실제 수업 배치를 관리하는 시트다.

가칭: `수업배치`

권장 컬럼:

- `slot_date`
- `slot_period`
- `slot_order`
- `과목`
- `lesson_id`
- `status`
- `source`
- `memo`

설명:

- `slot_date`: 수업 날짜
- `slot_period`: 교시
- `slot_order`: 같은 날짜, 같은 과목, 같은 교시 안에서의 안정적인 정렬값
- `과목`: 시간표 기준 과목
- `lesson_id`: 진도표 행과 연결되는 키
- `status`: `planned`, `done`, `skipped` 같은 슬롯 상태
- `source`: `auto`, `manual` 같은 생성 경로
- `memo`: 예외 조정 기록

### 3. 규칙 시트

기존 시트를 유지한다.

- `기초시간표`
- `휴업일`
- `수업시작일`

이 시트들은 브릿지 시트를 자동 생성하거나 재생성할 때만 사용한다.

## 데이터 책임

### 진실 원천

- 수업 내용과 순서의 진실 원천: `진도표`
- 날짜/교시 배치의 진실 원천: `수업배치`
- 자동 생성 규칙의 진실 원천: `기초시간표`, `휴업일`, `수업시작일`

### 파생값

- `진도표.계획일`
- `진도표.계획교시`

이 두 값은 브릿지 시트에서 계산해 동기화하는 표시용 값으로 본다.

## 편집 원칙

### 진도표에서 하는 일

- 수업한 항목 체크
- 수업 순서 조정
- 수업 추가/삭제
- 수업 내용 수정

### 브릿지 시트에서 하는 일

- 특정 날짜에 어떤 과목을 배치할지 확인
- 날짜 이동
- 교시 이동
- 특정 슬롯 비우기 또는 수동 연결

### 하지 않을 일

- 진도표의 `계획일`을 사람이 직접 수정하지 않는다.
- 브릿지 시트에 수업 본문을 중복 저장하지 않는다.

## 동기화 규칙

### 브릿지 -> 진도표

브릿지 시트를 기준으로 각 `lesson_id`의 가장 이른 예정 슬롯을 찾아 진도표의 표시 컬럼에 반영한다.

- `계획일`: 가장 이른 `planned` 또는 `done` 슬롯의 날짜
- `계획교시`: 해당 슬롯의 교시

### 진도표 -> 브릿지

진도표의 순서가 바뀌거나 새 수업이 생기면 자동 배치 프로그램이 이를 읽어 브릿지 시트의 `lesson_id` 연결을 다시 계산한다.

수동 이동된 슬롯은 `source=manual` 또는 별도 플래그를 두어 재배치 시 보존 여부를 결정한다.

## 최소 마이그레이션 순서

### 1단계

- 설계 문서 고정
- `lesson_id` 도입 방안 확정

### 2단계

- 진도표에 `lesson_id` 컬럼 추가
- 기존 행에 누락 없는 ID 부여
- 코드에서 `_row` 대신 `lesson_id`를 우선 사용하도록 준비

### 3단계

- `수업배치` 시트 읽기/쓰기 모듈 추가
- 기존 자동 배치 로직이 `계획일` 대신 브릿지 시트를 생성하도록 변경

### 4단계

- 브릿지 시트 결과를 진도표 `계획일`, `계획교시`에 동기화
- 기존 조회/완료/밀기/연장 기능을 브릿지 기준으로 순차 전환

### 5단계

- UI에서 날짜 보기와 진도 보기 역할을 분리
- 필요 시 `계획일` 직접 수정 경로 제거

## 구현 시 주의점

- 현재 코드와 UI는 시트 행번호 `_row` 의존도가 높다.
- 행번호는 삽입/정렬에 취약하므로 브릿지 연결 키로 쓰지 않는다.
- 같은 과목이 같은 날 여러 번 있는 경우를 위해 `교시`와 `slot_order`를 함께 둔다.
- 완료 처리는 가능한 한 브릿지 슬롯과 진도표 상태를 함께 갱신해야 한다.

## 이번 설계의 원칙

- 기존 화면을 한 번에 깨지 않도록 `계획일` 컬럼은 당분간 유지한다.
- 새 구조의 기준 데이터는 브릿지 시트로 이동한다.
- 큰 전환은 한 번에 하지 않고, 호환층을 두고 점진적으로 바꾼다.
